Data Visualization

It's no secret that data can be very powerful — when you can actually understand what it's telling you, that is.

It's not easy to get clear takeaways by looking at a slew of numbers and stats. You need to have the data presented in a logical, easy-to-understand way so that you can apply your learnings in an effective way. Using charts, graphs, and design elements, data visualization can help you explain trends and stats much more easily. But, not all data visualization is created equal.

Real-Life Example

Here is a hypothetical example of a chart that was used in a report.

We are testing three units of the same product that has Wi-Fi and each has 2 outputs. We want to know the transmit power vs channel number of each of the outputs, if they are within set limits and the differences between the output and the individual units.

At a quick glance, we can get an idea that some outputs have higher power than others and it takes a bit more examination to try and understand the differences between outputs or units but it's not very clear. We also can not see if the results pass or fail.

By modifying how the chart plots to include product ID and adding the limits we can instantly pick out more value from the same data.

Now we are able to see that unit ID_003 fails on both outputs and that output B is higher than output A in nearly all cases. Reporting this to the designer will help them to investigate the causes and improve the design.

Viewing Multi-Dimensional Data

When we're trying to view trends of multiple variables or large data sets it can be difficult to visualise on two dimensional plots.

Multi-panel plots allows us to see both distribution of single variables and relationships between two variables.

Multi-panel Plot Of Bluetooth Sample Results.

The above plot shows the distribution for each "result value" -- dBm, kHz, and ratio -- across a range of three different temperatures. The plot is further divided into three columns, which is the "data packet type" (don't worry too much about the technical jargon.)

Based on the above plot, you can see that plot collections are a useful way of observing potential relationships or patterns in two-dimensions for data attributes. Here we can see that temperature affects the results in the first set of tests but data packet type does not. We would then drill down into the data to see how each unit is affected and this would give us insights to compensate for temperature in the design.
